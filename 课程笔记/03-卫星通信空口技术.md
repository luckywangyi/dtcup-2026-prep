# 03 卫星通信空口技术

> 对应学唐平台课程：卫星通信空口技术

## 核心知识点

### 一、NTN空口技术挑战

卫星通信相比地面通信面临的特殊挑战：

| 挑战 | 原因 | 影响 |
|------|------|------|
| **长传输时延** | 卫星高度导致（GEO约270ms单程） | HARQ、定时器等协议需适配 |
| **大多普勒频移** | 卫星与地面相对运动（尤其LEO） | 频率偏移导致解调困难 |
| **大路径损耗** | 传播距离远 | 需要更大发射功率/增益天线 |
| **波束切换** | LEO卫星移动导致波束变化 | 需要频繁切换 |
| **覆盖区域变化** | LEO非静止 | 小区拓扑随时变化 |

### 二、时延补偿技术

#### 定时提前（Timing Advance）

- 地面5G中TA由基站通过RAR消息告知UE
- NTN中由于传播时延极大，需要**UE预补偿**
- UE通过**星历信息（ephemeris）**+自身GNSS位置计算传播时延
- 预补偿后的残余时延由基站TA命令微调

#### HARQ适配

- 地面5G：HARQ反馈通常在几ms内
- NTN：由于往返时延大，传统同步HARQ不可行
- 解决方案：**禁用HARQ反馈** 或 **增大HARQ进程数**

### 三、多普勒补偿技术

#### 多普勒频移来源

- LEO卫星移动速度约7.5 km/s
- 最大多普勒频移可达 **±24 ppm**（取决于轨道高度和频段）

#### 补偿策略

**发射端预补偿：**
- 通过卫星星历和载波频率计算多普勒频移
- 以slot为单位对OFDM信号进行频率预补偿
- UE需要知道卫星位置（通过星历或GNSS）

**接收端补偿：**
- **独立时频同步**：先补偿多普勒频移，再补偿定时偏移
- **联合时频同步**：同时补偿频偏和时偏，利用DM-RS参考信号估计

### 四、NTN波束管理

#### 波束类型

| 类型 | 说明 |
|------|------|
| **固定波束** | 波束指向固定区域（地面固定点），卫星移动时波束跟踪 |
| **移动波束** | 波束随卫星移动，覆盖区域随之变化 |

#### 波束切换

- LEO卫星移动导致波束覆盖区域不断变化
- 系统通过卫星可预测的运动轨迹（星历）进行波束切换规划
- 类似地面网络的小区切换，但触发条件基于**位置/时间**而非信号质量

### 五、NTN物理层适配

#### 子载波间距

- NTN通常使用较大的子载波间距（如30kHz/60kHz）
- 较大SCS提高对多普勒频移的容忍度

#### PRACH适配

- NTN需要加长PRACH前导码（preamble）以覆盖往返时延
- R17定义了专门的NTN PRACH格式

#### 功率控制

- 考虑卫星高度导致的大路径损耗
- 上行功率控制需要补偿自由空间损耗
- UE最大发射功率可能不足，需依赖增益天线

### 六、NTN移动性管理

#### 卫星间切换（Inter-satellite Handover）

- LEO卫星过境时间短（约10分钟）
- 需要频繁的卫星间切换
- 切换基于卫星星历可预测规划

#### 地面-卫星切换

- UE在地面网络覆盖区域优先使用地面网络
- 离开地面覆盖区域时切换到卫星网络
- 实现星地一体无缝覆盖

## 考点速记

| 考点 | 记忆要点 |
|------|---------|
| NTN五大挑战 | 时延/多普勒/路径损耗/波束切换/覆盖变化 |
| 时延预补偿 | UE用星历+GNSS自主计算 |
| HARQ适配 | 禁用反馈或增大进程数 |
| LEO速度 | ~7.5km/s |
| 多普勒预补偿 | 发射端基于星历频率预偏 |
| 波束切换 | 基于位置/时间，非信号质量 |
| SCS选择 | 大SCS提高多普勒容忍度 |
| LEO过境时间 | ~10分钟 |
