## 一、卫星网络架构介绍

### 1.1 卫星组网特征

卫星网络既可单独部署，也可作为地面网络的补充。

| 特征 | 说明 |
|------|------|
| **全球/区域覆盖** | 单颗GEO覆盖地球表面约1/3区域；多颗LEO星座组网可实现全球无缝覆盖 |
| **非静止轨道高速移动** | LEO卫星相对地面高速移动（速度约**7.8km/s**），网络拓扑实时变化，需动态路由和切换技术 |
| **传输时延差异大** | GEO信号需往返约**7.2万公里**，时延约**250~400ms**，不适合实时交互应用（如视频通话）；LEO轨道高度500~2000km，时延可降至**20~50ms**，接近地面光纤 |
| **星间链路（ISL）** | 星座卫星间直接通信（如"星链"），减少对地面站依赖，但需复杂星上处理能力 |
| **传输距离远** | 卫星作为中继站，可跨越数千公里直接传输信号，无需地面中继设施，适合跨洲际通信 |
| **多段协同** | 涉及空间段（卫星）、地面段（信关站、控制中心）、用户段（终端设备）三段协同 |
| **共享带宽** | 用户需竞争使用信道，高并发场景下可能拥塞 |
| **抗灾能力强** | 不受地面自然灾害（地震、洪水）或人为破坏影响，适合应急通信（如救灾、军事） |
| **偏远地区通信** | 适用于海洋、沙漠、高山等地面基站难以覆盖的场景，为科考、航海、航空等提供通信保障 |

### 1.2 卫星网络整体结构

卫星互联网系统由**一个网络、两个平面**组成：

```

                    │       卫 星 平 面        │
                    │  LEO(T) LEO(T) LEO(T)   │ ← 透明转发卫星
                    │  LEO(R) LEO(R) LEO(R)   │ ← 再生转发卫星
                    └───────────┬─────────────┘
                                │
              ┌─────────────────┼─────────────────┐
              │                 │                 │
      ┌───────┴────┐    ┌──────┴──────┐   ┌──────┴──────┐
      │   射频      │    │  NTN核心网   │   │  网管        │
      │  主站/gNB   │    │             │   │             │
      └──────┬─────┘    └──────┬──────┘   └─────────────┘
             │                 │
      ┌──────┴─────────────────┴──────┐
      │        地面核心网               │
      └──────┬────────────────────────┘
             │
      ┌──────┴─────┐
      │  地面基站平面 │
      └────────────┘
```

**五大融合特征**：

| 融合维度 | 说明 |
|---------|------|
| **终端融合** | 统一终端标识与接入方式 |
| **体制融合** | 卫星与地面蜂窝通信体制统一 |
| **系统融合** | 网络平台采用一体化设计 |
| **网络融合** | 全网统一网络架构 |
| **业务融合** | 统一业务支持与调度 |

### 1.3 关键链路定义

| 链路 | 英文 | 两端 | 说明 |
|------|------|------|------|
| **服务链路** | Service Link | 卫星 ↔ UE | 直接面向用户终端的无线链路 |
| **馈电链路** | Feeder Link | 卫星 ↔ 信关站/网关 | 卫星与地面网络之间的回传链路，无ISL时为必须链路 |
| **星间链路** | ISL (Inter-Satellite Link) | 卫星 ↔ 卫星 | 星座内卫星间直接通信，减少对地面站依赖 |

### 1.4 两种负荷处理模式

#### 透明转发（Bent-Pipe）

- 卫星**仅具备**射频滤波、频率转换和放大功能
- 卫星就是一个RF中继器，不解调不解码
- 5G gNB功能**全部在地面**实现（gNB在NTN Gateway侧）
- **有效载荷（Payload）**：转发器、天线 = 卫星有效负荷
- **3GPP R17** 标准主要支持方案

```
透明转发 NG-RAN 架构：

UE ←─NR-Uu─→ Satellite ←─NR-Uu─→ gNB(NTN Gateway) ─NG─→ 5G CN ─N6─→ DN
               (RF中继)          (基带处理在地面)
```

#### 再生转发（Non-Transparent / On Board Processor）

- 除了射频滤波、频率转换和放大功能外，还具备**调制/编码、解调/解码、交换/路由**等功能
- 卫星搭载gNB（部分或全部），具备完整基带处理能力
- 支持**星间链路ISL**，实现空间组网
- **3GPP R18+** 推进

```
再生转发 NG-RAN 架构（无ISL）：

UE ←─NR-Uu─→ gNB(卫星上) ─NG over SRI─→ NTN Gateway ─NG─→ 5G CN ─N6─→ DN

再生转发 NG-RAN 架构（有ISL）：

UE ←NR-Uu→ gNB(卫星A) ←─Xn(over ISL)─→ gNB(卫星B) →NTN GW→ 5G CN → DN
                              ↑ 星间链路
```

#### 对比速查表

| 对比项 | 透明转发 | 再生转发 |
|--------|---------|---------|
| 卫星处理 | 仅RF（滤波+频率转换+放大） | RF + 基带（调制编码/解调解码/路由） |
| gNB位置 | **地面**（NTN Gateway侧） | **星上**（部分或全部） |
| 有效载荷 | 转发器、天线 | 星载gNB、路由器 |
| ISL支持 | 不支持 | 支持 |
| 复杂度 | 低 | 高 |
| 灵活性 | 低 | 高 |
| 3GPP版本 | **R17** | **R18+** |
| 类比 | 镜子（只反射） | 中转站（能拆包处理） |

### 1.8 卫星部署演进路线

中信科移动给出四阶段演进路线，逐步将功能从地面推向星上：

| 阶段 | 方式 | 架构说明 |
|------|------|---------|
| **方式一** | 卫星透明转发 | UE ↔ 透明卫星 ↔ 信关站(gNB) → NR CN；gNB完全在地面 |
| **方式二** | 仅DU上星 | UE ↔ gNB-DU(星上) ↔ F1-over-SRI ↔ 信关站(gNB-CU) → NR CN；CU/DU分离 |
| **方式三** | gNB上星 | UE ↔ gNB(星上) ↔ NG-over-SRI ↔ 信关站 → NR CN；完整基站上星 |
| **下阶段** | 核心网上星及演进 | UE ↔ gNB(星上) ↔ NR CN(星上) ↔ 信关站；核心网功能也部署到卫星 |

对应3GPP标准演进：
- **R15~R18**：NTN透明转发（方式一）
- **R18**：基站上天（方式二/三）
- **R19+**：核心网上天（下阶段）

### 1.9 星地融合网络

#### 网络组成

| 网络层 | 组成 | 说明 |
|--------|------|------|
| **天基网络** | 高轨GEO + 中低轨MEO/LEO | 包括天基接入网和天基核心网 |
| **地基网络** | 地面核心网 + 地面基站 | 地面核心网可细分为公众核心网和卫星核心网 |

#### 应用场景覆盖

天基网络覆盖：民航客机、无人机、ATG（空对地）、物联网、定位导航、气象通信、偏远山区通信、农村地区、城镇地区、车联网、近海地区、大型商船、渔船、境外通信、海洋地区

#### 网络融合方式三阶段

| 阶段 | 名称 | 说明 |
|------|------|------|
| 第一阶段 | **互联互通**（业务融合） | 星地独立网络，卫星通信体制借鉴5G |
| 第二阶段 | **混合接入**（体制融合） | 部分共享网络资源 |
| 第三阶段 | **深度融合** | 星地一体，提供无感知一致服务 |

#### 面向6G的系统融合（七大融合）

| 融合维度 | 含义 |
|---------|------|
| 体制融合 | 统一空口传输 |
| 网络融合 | 全网统一网络架构 |
| 管理融合 | 统一资源调度与管理 |
| 频谱融合 | 频谱共享共用、协调管理 |
| 业务融合 | 统一业务支持与调度 |
| 平台融合 | 网络平台采用一体化设计 |
| 终端融合 | 统一终端标识与接入方式 |

### 1.10 星地融合组网关键技术

| 技术领域 | 研究方向 |
|---------|---------|
| **网络架构** | 卫星与地面蜂窝通信架构统一设计；弹性可重构灵活网络架构；星地节点间网络功能柔性分割 |
| **空口传输** | 统一空口设计方案；支持多种业务传输；终端极致简单接入到最合适星地网络节点 |

| **频率管理** | 基于统一管理的网络频谱资源，星地间频谱协调管理机制；频谱共享和干扰管理方案 |

## 二、星地、网间关键接口定义与功能

### 2.2 卫星专网系统功能

#### 无线接入网（Ka频段）

| 接入方式 | 组成 | 转发模式 |
|---------|------|---------|
| **低轨Ka频段无线接入网** | 卫星终端 + 星载基站 | 星上处理转发；支持随遇波束接入和业务波束接入；单星或多星组网 |
| **高轨Ka频段无线接入网** | 卫星终端 + 星载透明/再生转发器 + 地面高轨基站 | 透明转发：信号透传到地面高轨基站完成接入；再生处理：星载再生转发器和地面基站联合完成接入 |

#### 天基承载网

天基承载网是实现卫星互联网空间卫星网络互联、高效可靠传输和传输质量保证的系统。

组成：
- **星载路由器**：部署于卫星上的路由交换设备
- **星间链路载荷**（激光星间链路）
- **空间网络控制中心**（部署在地面运行控制中心，类似地面网络控制器）
- 星地路由（馈电侧）+ 地面网络控制器 + 信关站

#### 核心网

核心网基于**5GC和IMS**设计改造，包括地面核心网部分和星载S-UPF网元（低轨）。

| 功能模块 | 说明 |
|---------|------|
| **地面核心网** | 包括AMF、SMF、UPF等系列网元，为高低轨卫星Ka频段宽带统一提供服务开通、认证鉴权、用户管理、计费等功能 |
| **核心网与接入网关** | 核心网与接入网关/地面网关连接，完成天地协同处理，考虑到系统对组网架构等需求，在低轨卫星部署S-UPF，以支持5G-LAN功能的星上实现 |
| **IMS域** | IMS可为点对点、点对多点业务提供多媒体通道的控制与建立；IMS具有统一的QoS和计费策略控制机制；IMS通过MGCF/MGW或IBCF/TrGW与地面运营商网络实现语音互联互通 |

### 2.3 高轨卫星通信系统

#### 高轨通信系统组成

```
 高轨透明转发卫星 ←─G-ISL星间链路─→ 高轨再生卫星 ←─G-ISL─→ 高轨透明转发卫星
      │                                │                        │
   用户链路                          用户链路                   用户链路
      │                                │                        │
   卫星终端              馈电链路       │      馈电链路          卫星终端
                           │           │         │
                        信关站    核心网+网控系统  信关站
```

#### 高轨卫星通信链路

| 链路路径 | 频段 | 调制方式 |
|---------|------|---------|
| 用户终端 ↔ 卫星（处理用户） | **Ka频段** | OFDM |
| 卫星 ↔ 信关站（透传馈电） | **Ka频段** | 单载波 |
| 卫星 ↔ 卫星（星间） | **激光** | — |
| 信关站 ↔ 卫星（馈电/业务） | **Q/V频段** | OFDM / 单载波 |

#### 高轨卫星功能划分

高轨系统按照网络功能划分为三部分：**无线接入网 + 天基承载网 + 核心网**

| 功能域 | 包含组件 |
|--------|---------|
| **无线接入网** | 卫星载荷（透明/再生） + 固定终端 + 高轨卫星终端 + 接入请求波束（透明/处理） + 用户波束（处理/星载） |
| **天基承载网** | 高轨卫星 + 星载路由器 + 协议网关 + 高轨透传基站（地面） |
| **核心网** | 5GC域（AMF/SMF/UPF/AUSF/AF...）+ IMS域 + CS域 + 天基承载网网络控制器 |

#### 高轨系统网元

| 序号 | 网元名称 | 功能 |
|------|---------|------|
| 1 | **UE** | 卫星终端信号发送/接收设备，完成所需业务功能和接入卫星通信网络等多方面任务 |
| 2 | **G-gNB-AN / G-gNB** | 提供用户接入、无线资源管理、终端数据传输服务 |
| 3 | **G-gNB-HL / S-MAC** | 提供再生模式卫星的除物理层之外的高层协议、接口协议的处理 |
| 4 | **G-gNB-PHY / S-PHY** | 提供再生模式卫星的物理层处理 |
| 5 | **S-Feeder** | 维持和信关站之间的馈电链路，并进行数据收发 |
| 6 | **G-Feeder** | （地面侧馈电链路端） |

## 四、考点速记

| 考点 | 记忆要点 |
|------|---------|
| NTN组成 | 卫星 + 网关(信关站) + UE + 核心网 |
| Service Link | 卫星 ↔ UE（服务链路） |
| Feeder Link | 卫星 ↔ 信关站/网关（馈电链路），无ISL时必须 |
| ISL | 卫星 ↔ 卫星（星间链路） |
| 透明转发定义 | **仅**具备射频滤波、频率转换和放大功能 |
| 再生转发定义 | 还具备调制/编码、解调/解码、交换/路由等功能 |
| 透明转发gNB位置 | **地面**（NTN Gateway侧） |
| 再生转发gNB位置 | **星上** |
| 有效载荷 | 转发器、天线 = 卫星有效负荷 |
| 部署演进四阶段 | 透明转发 → DU上星 → gNB上星 → 核心网上星 |
| NTN标准起始 | **R15** 开始研究（2018年） |
| 第一个NTN标准 | **R17**（2022年） |
| 再生转发标准 | **R18+** |
| LEO速度 | 约 **7.8 km/s** |
| GEO往返距离/时延 | 约 7.2万公里 / **250~400ms** |
| LEO时延 | **20~50ms** |
| 五大融合 | 终端、体制、系统、网络、业务 |
| 6G七大融合 | 体制、网络、管理、频谱、业务、平台、终端 |
| 一个网络两个平面 | 一个网络（卫星互联网），两个平面（卫星平面+地面基站平面） |
| 卫星专网四大段 | 空间段 + 地面段 + 应用段 + 外部网络 |
| 核心网组成 | 5GC + IMS 双域设计 |
| S-N2接口 | 基站与AMF的接口，也叫S-NG-C |
| S-N3接口 | 基站与UPF的接口，也叫S-NG-U |
| S-ISL接口 | 卫星间路由和切换信息 |
| 中国信科角色 | 牵头完成信关站、核心网、终端模拟器、多终端模拟器及星载基站 |
