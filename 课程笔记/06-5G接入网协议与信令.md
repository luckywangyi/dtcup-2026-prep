# 06 5G接入网协议与信令

> 对应学唐平台课程：10 DTM.PX4.016.007-5G接入网协议与信令

## 核心知识点

### 一、系统消息体系

#### MIB vs SIB1

| | MIB | SIB1 |
|---|-----|------|
| 承载信道 | PBCH（在SSB中） | PDSCH |
| 广播方式 | **周期广播** | **周期广播** |
| 主要内容 | SFN、SCS配置、**如何获取SIB1** | RACH配置、TDD配置、其他SIB调度 |

- 最小系统消息 = MIB + SIB1
- MIB一个最重要作用是通知UE如何获取 **SIB1**
- MIB包含：系统帧号(SFN) + SIB1的PDCCH CORESET配置
- MIB **不包含**：小区ID（由PSS/SSS获取）、PHICH配置（LTE概念）
- OSI是采用周期广播还是订阅由 **SIB1** 通知
- SIB2主要包含：小区重选系统内异频/同频相关信息、小区异系统重选公共消息

### 二、随机接入流程（RACH）

#### 四步竞争随机接入

```
UE                        gNB
 |== Msg1: Preamble ===→  |  (PRACH)
 |←= Msg2: RAR =========  |  (PDSCH, 含TA+TC-RNTI+UL Grant)
 |== Msg3: RRC Setup Req →|  (PUSCH)
 |←= Msg4: 竞争解决 =====  |  (PDSCH)
```

#### 竞争 vs 非竞争随机接入

| 场景 | 随机接入类型 |
|------|-------------|
| **初始接入（IDLE态）** | **竞争** |
| **无线链路失败后初始接入** | **竞争** |
| **上行失步+无SR资源** | **竞争** |
| **切换** | **可竞争可非竞争** |
| **波束失败恢复** | **非竞争** |

### 三、RRC状态

| 状态 | 说明 |
|------|------|
| **RRC_IDLE** | 空闲态，无连接 |
| **RRC_INACTIVE** | 非活跃态（5G新增），保留上下文 |
| **RRC_CONNECTED** | 连接态 |

- RM DEREGISTERED / RM REGISTERED 是**NAS层**状态，不是RRC状态
- RRC INACTIVE恢复业务触发信令：**RRC resume request**
- RRC重建触发原因：切换失败、重配失败、完保失败、基站检测RLF（全选）
- RRC重建时若UE上下文可恢复→RRC Resume；不能恢复但有资源→**RRC Setup**；拒绝→**RRC Reject**

### 四、NAS注册与初始接入信令

- 基站向核心网发的第一条消息：**Initial UE Message**
- NSA中携带SCG Add或PSCell Change配置的信令：**LTE侧RRCConnectionReconfiguration**
- NSA场景SRB3未建立时，SCG测量结果通过 **UL Information Transfer MRDC** 上报

### 五、SRB与DRB

- SRB0 → CCCH（公共控制信道）
- SRB1 → DCCH（RRC+NAS消息）
- SRB2 → DCCH（仅NAS消息）
- EN-DC下可建立 SRB0/SRB1/SRB2/SRB3（全选）
- QoS Flow与DRB：多个QoS Flow可映射一个DRB；一个QoS Flow映射一个DRB；但一个QoS Flow**不能**映射多个DRB

### 六、寻呼

- PO（Paging Occasion）是一套PDCCH监听机会
- 一个PF里可有多个PO
- 一个PO的长度等于一个波束扫描周期
- 在每个波束上发送的Paging消息是**一样的**（不是不一样）

### 七、信令流程（仿真重点）

#### 七大信令流程

| 序号 | 流程 | 关键考点 |
|------|------|----------|
| 1 | 系统消息获取 | PSS/SSS→MIB→SIB1 |
| 2 | 随机接入+RRC连接 | 四步RACH：Msg1-4 |
| 3 | NAS鉴权加密 | Registration Request→Authentication |
| 4 | UE能力询问+AS加密 | SecurityModeCommand |
| 5 | RRC重配 | RRCReconfiguration |
| 6 | PDU Session建立 | DRB建立+QoS配置 |
| 7 | 测量切换 | A3事件→HandoverRequest |

#### 切换信令

**Xn切换流程：**
```
源gNB → 目标gNB: Handover Request（第一条信令）
目标gNB → 源gNB: Handover Request Acknowledge
源gNB → UE: RRCReconfiguration
UE → 目标gNB: 随机接入 + RRCReconfigurationComplete
目标gNB → AMF: Path Switch Request
```

**NG切换流程：**
- 切换请求经AMF转发
- Handover Required (源→AMF) → Handover Request (AMF→目标)

### 八、其他

- 5G缩短时延的关键技术：RRC-inactive状态、MEC网络架构、自包含帧结构、Shorter-TTI（全选）
- T300定时器：发送RRC Setup Request时**启动**，接收到RRC Setup或RRC Reject后**停止**

## 考点速记

| 考点 | 记忆要点 |
|------|---------|
| MIB作用 | 通知UE如何获取SIB1 |
| 最小系统消息 | MIB + SIB1 |
| MIB不含 | 小区ID、PHICH |
| INACTIVE恢复信令 | RRC resume request |
| 基站→核心网第一条 | Initial UE Message |
| NSA携带SCG配置 | LTE侧RRCConnectionReconfiguration |
| RRC状态 | IDLE/INACTIVE/CONNECTED（不含RM状态） |
| RRC重建原因 | 切换失败/重配失败/完保失败/RLF |
| 竞争随机接入 | 初始接入/RLF后/无SR资源 |
| 非竞争随机接入 | 波束恢复/切换（切换也可竞争） |
| Xn切换第一条 | Handover Request |
| QoS Flow→DRB | 多对一或一对一，不可一对多 |

## 相关真题

> **单选**：MIB一个最重要作用是通知UE获取？→ **SIB1** [第九届A/B组]

> **多选**：5G系统中哪些系统消息一定是周期广播？→ **MIB和SIB1** [第十届A组]

> **多选**：MIB消息中包含哪些内容？→ **系统帧号 + SIB1的PDCCH CORESET配置** [第十届A组]

> **单选**：RRC INACTIVE恢复业务触发哪条信令？→ **RRC resume request** [第九届A/B组、第十届A组]

> **多选**：5G NR中RRC状态包括？→ **CONNECTED/IDLE/INACTIVE** [第九届A/B组]

> **多选**：哪种情形下只能进行竞争随机接入？→ **RLF后初始接入、IDLE态初始接入、无SR资源** [第九届A组]

> **判断**：切换过程的随机接入可以采用非竞争也可以竞争 → **正确** [第十届A组]

> **单选**：基站向核心网发的第一条消息？→ **Initial UE Message** [第九届A/B组]

> **多选**：5G RRC连接重建触发原因？→ **全选：切换失败/重配失败/完保失败/RLF** [第九届B组]

> **多选**：缩短时延的关键技术？→ **全选：RRC-inactive/MEC/自包含帧/Shorter-TTI** [第九届B组]

> **多选**：EN-DC下可建立哪些SRB？→ **全选：SRB0/SRB1/SRB2/SRB3** [第九届B组]

> **多选**：QoS Flow与DRB关系正确的是？→ **多对一/一对一/DRB保证QoS** [第九届B组]

> **多选**：T300定时器说法正确的是？→ **发送RRC Setup Request时启动；收到RRC Setup或Reject时停止** [第十届A组]
