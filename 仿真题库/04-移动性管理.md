# 仿真实验04：移动性管理

> 参考视频：不想做仿真的熊二、J_K_Wang、-70L 移动性管理系列

## 实验概述

本实验考查5G NR中的小区选择与重选、测量与切换流程。

## 核心知识点

### 小区选择与重选

#### 小区选择（Cell Selection）

UE开机后选择驻留小区的过程：
1. 频率搜索 → 检测PSS/SSS
2. 读取MIB/SIB1
3. 判断小区是否满足驻留条件：
   - PLMN匹配
   - cellBarred = notBarred
   - RSRP ≥ q-RxLevMin

#### 小区重选（Cell Reselection）

UE在IDLE/INACTIVE态更换驻留小区：
- **同频重选**：基于SIB2中的参数，Ranking准则
- **异频重选**：基于优先级，高优先级绝对优先
- **异系统重选**：4G↔5G重选

### 5G测量事件

| 事件 | 触发条件 | 用途 |
|------|---------|------|
| **A1** | 服务小区好于门限 | 停止异频测量 |
| **A2** | 服务小区差于门限 | 启动异频测量 |
| **A3** | 邻区好于服务小区+偏移 | **同频切换** |
| **A4** | 邻区好于门限 | 异频切换 |
| **A5** | 服务小区差于门限1且邻区好于门限2 | 异频/异系统切换 |
| **B1** | 异系统邻区好于门限 | 异系统切换 |

### 切换流程

#### 基站内切换（Intra-gNB Handover）

同一gNB内小区间切换，不涉及核心网。

#### Xn切换

```
1. 源gNB → 目标gNB: Handover Request
2. 目标gNB → 源gNB: Handover Request Acknowledge
3. 源gNB → UE: RRCReconfiguration（含目标小区配置）
4. UE → 目标gNB: 随机接入
5. UE → 目标gNB: RRCReconfigurationComplete
6. 目标gNB → AMF: Path Switch Request
7. AMF → 目标gNB: Path Switch Request Acknowledge
```

#### NG切换

经核心网AMF转发：
```
1. 源gNB → AMF: Handover Required
2. AMF → 目标gNB: Handover Request
3. 目标gNB → AMF: Handover Request Acknowledge
4. AMF → 源gNB: Handover Command
5. 后续同Xn切换
```

### 邻区配置

- NR邻基站配置需要配置：目标gNB ID、IP地址等
- PCI规划：PCI = 3×N_ID_1 + N_ID_2，同站三扇区mod3不同

## 仿真操作要点

1. 配置邻区关系（添加NR邻基站）
2. 配置测量事件（A3事件参数：偏移量、迟滞、TTT）
3. 观察切换信令流程
4. 验证切换后UE的PCI变化
5. 注意Xn切换和NG切换的区别
