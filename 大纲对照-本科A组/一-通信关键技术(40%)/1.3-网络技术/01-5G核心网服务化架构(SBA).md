> 大纲分类：一、通信关键技术 > 三、网络技术 > 5G核心网服务化架构(SBA)  
> 考核要求：掌握

## 核心知识点

### 一、SBA（Service Based Architecture）概念

**服务化架构（SBA）** 是 5G 核心网（5GC）的基础设计范式：将传统“网元—网元”的**点对点信令互联**，改为各 **网络功能（NF, Network Function）** 以**可复用的服务（Service）**形式对外暴露能力；其他 NF 作为**消费者**通过标准 API **发现、选择并调用**这些服务。

典型技术特征包括：

- **接口统一**：服务化接口多基于 **HTTP/2 + JSON**，配合 **OpenAPI** 描述，便于自动化与云原生集成。  
- **松耦合**：功能模块可独立扩缩容、独立升级（灰度/多版本并存）。  
- **可编排**：结合切片、边缘部署时，可按业务链灵活组合 NF 服务实例。

### 二、与传统核心网“点对点网元架构”的区别

| 对比维度 | 传统 EPC 等点对点架构 | 5GC 服务化架构（SBA） |
|---------|----------------------|------------------------|
| 互联方式 | 网元间固定接口、拓扑耦合强 | NF 间通过“服务调用”，逻辑拓扑灵活 |
| 能力暴露 | 以“接口/协议栈”为中心 | 以“服务/API”为中心 |
| 发现与选择 | 多依赖静态配置或间接机制 | **NRF** 支持注册、发现、健康与授权相关机制 |
| 扩展性 | 新增功能常牵动多接口 | 新增 Producer/Consumer 相对独立 |
| 运维与自动化 | 配置与对接成本高 | 利于 DevOps、API 网关、服务网格治理 |

**易错点（与真题一致）**：SBA 强调**移动性与会话管理解耦**、**控制与用户面分离**、**多种接入统一接入机制**；**不是**“继续采用传统点对点通讯方式作为核心特征”。（见下方真题：研究生组多选题 120，D 为干扰项。）

### 三、NF 服务化接口与命名规则

每个 NF 可对外提供多类服务；**服务化接口名称**通常记为 **N&lt;NF&gt;**，例如：

| 服务化接口 | 典型服务端 NF | 说明（示例服务，非穷举） |
|-----------|--------------|---------------------------|
| **Namf** | AMF | 接入与移动性管理相关服务（如 Namf_Communication） |
| **Nsmf** | SMF | 会话管理、PDU 会话相关服务 |
| **Npcf** | PCF | 策略控制 |
| **Nudm** | UDM | 签约数据、订阅管理 |
| **Nausf** | AUSF | 鉴权服务 |
| **Nnrf** | NRF | NF 注册、发现、状态等 |
| **Nnssf** | NSSF | 网络切片选择 |
| **Nnef** | NEF | 能力开放 |
| **Nupf** | UPF | 用户面相关（UPF 也参与部分服务化交互场景，具体以规范为准） |

**非服务化参考点接口**（仍需掌握概念）：如 **N1**（UE—AMF）、**N2**（RAN—AMF）、**N3**（RAN—UPF）、**N4**（SMF—UPF）、**N6**（UPF—DN）等，用于区分“空口/接入/用户面转发平面”与“核心网内部服务化平面”。

### 四、NRF 与服务注册 / 发现 / 调用机制

**NRF（Network Repository Function，网络存储库功能）** 是服务化架构的“目录与治理中枢”之一，主要承担：

1. **注册（Register）**：NF 实例向 NRF 登记自身身份、支持的服务名称、可达地址、实例 ID 等。  
2. **更新 / 去注册**：实例生命周期变化时维护目录一致性。  
3. **发现（Discovery）**：消费者 NF 按服务名、切片、DNN、位置等条件查询合适实例。  
4. **辅助安全与策略**：可与安全架构配合完成授权、令牌校验等（具体行为以 3GPP 安全相关章节为准）。

**调用路径（概念级）**：Consumer →（可选）查 NRF 得 Producer 地址 → **HTTP 服务调用** → 业务处理。

### 五、5GC 主要网元职责速览

| 网元 | 英文全称（记忆用） | 核心职责（掌握级概括） |
|------|-------------------|------------------------|
| **AMF** | Access and Mobility Management Function | 接入、移动性、注册连接管理；NAS 终结于 AMF（常见考点） |
| **SMF** | Session Management Function | PDU 会话建立/修改/释放；与 UPF 交互；IP 分配等（**NAS 加密完整性不属于 SMF**，属 AMF） |
| **UPF** | User Plane Function | 用户面转发、锚点、分流（与边缘/MEC 强相关） |
| **AUSF** | Authentication Server Function | 鉴权相关处理 |
| **UDM** | Unified Data Management | 用户标识、签约数据、订阅数据等 |
| **PCF** | Policy Control Function | 策略与计费控制规则 |
| **NEF** | Network Exposure Function | 对第三方安全开放网络能力 |
| **NSSF** | Network Slice Selection Function | 切片选择辅助 |
| **NRF** | Network Repository Function | NF 注册与发现 |
| **AF** | Application Function | 应用层功能实体，经 NEF 等与 5GC 交互 |

## 考点速记

| 考点 | 记忆要点 |
|------|---------|
| SBA 本质 | NF 提供服务、HTTP/2 调用、松耦合可扩展 |
| vs 传统 | 点对点固定拓扑 → 服务注册发现 + API |
| NRF | 注册/发现/更新/去注册；服务自动化关键 |
| 接口前缀 | Namf/Nsmf/Npcf/Nudm/Nausf/Nnrf/Nnssf/Nnef… |
| AMF | 移动性/接入/注册；**NAS（N1）MM 消息终结** |
| SMF | 会话管理、N4 对 UPF；**不负责 NAS 信令保护** |
| UPF | 用户面转发与锚点 |
| 易混判断 | SBA **不是**“仍采用点对点作为核心” |
