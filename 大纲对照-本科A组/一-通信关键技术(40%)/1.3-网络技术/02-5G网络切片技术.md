## 核心知识点

### 一、网络切片的概念与动机

**网络切片（Network Slicing）** 指在同一套物理基础设施上，通过虚拟化、隔离与编排，构建多张**逻辑上的端到端网络（切片实例）**，分别满足差异化 KPI（带宽、时延、可靠性、成本、安全域等）。

**动机简述**：

- 垂直行业需求差异极大（车联网、工业控制、高清视频、海量物联）。  
- 避免为每类业务单独建专网带来的**重复投资**；同时比“一张公网打天下”更易做**SLA 保障**。  
- 与 **5GC SBA、NFV、云原生编排** 协同，实现切片生命周期管理。

### 二、典型切片类型与 ITU/3GPP 场景映射（掌握级）

常见课堂上与 **eMBB / uRLLC / mMTC** 三类 5G 场景对应记忆：

| 类型 | 典型诉求 | 业务举例 |
|------|----------|----------|
| **eMBB** | 大带宽、高速率 | 4K/8K、AR/VR、热点大容量 |
| **uRLLC** | 低时延、高可靠 | 工业控制、远程操控、部分车联网安全类 |
| **mMTC** | 海量连接、低功耗（常与其他技术组合） | 传感采集、大量终端在线 |

**注意**：真实部署中一张切片可**定制** KPI，并不总是“一对一”贴标签；考试常考**概念与隔离**而非工程细则。

### 三、S-NSSAI：切片的“身份证号”（SST + SD）

**S-NSSAI（Single Network Slice Selection Assistance Information）** 用于终端与网络侧**选择/标识**切片，主要由：

- **SST（Slice/Service Type）**：切片/业务类型（标准取值见 3GPP 定义，如 eMBB、URLLC、MIoT 等枚举）。  
- **SD（Slice Differentiator）**：可选，用于在同一 SST 下区分不同切片实例/运营商内部切片。

UE 可签约**默认切片**与**允许切片列表**；PDU 会话建立时会携带/协商与切片相关的信息（与试题中 **PDU 会话属性含 S-NSSAI** 考点一致）。

### 四、NSSF 与切片选择流程（概念级）

**NSSF（Network Slice Selection Function）** 辅助确定服务于该 UE 的**网络切片实例集合**，典型输入包括：签约、本地策略、位置、支持的切片列表、AMF 能力等。

高层理解：**AMF 与 NSSF 等协作**完成“选哪个切片实例提供服务”，再结合 SMF/UPF 等完成会话与用户面路径。

### 五、端到端切片：RAN + 传输 + 核心网

**端到端（E2E）切片** 需多域协同：

1. **无线接入（RAN）**：通过 QoS、资源划分、CU/DU 部署与共享策略等实现差异化。  
2. **传输网**：FlexE/VPN/SR-TE 等承载隔离与 QoS（教学层掌握“需要传输侧保障”即可）。  
3. **核心网（5GC）**：控制面/用户面 NF 实例可按切片部署；与 **SBA** 组合做灵活编排。

### 六、切片隔离与编排管理

**隔离**可分为（由弱到强，考试常考“逻辑隔离”表述）：

- **逻辑隔离**：资源与转发路径在控制上分离，共享物理设施（**最常见表述**）。  
- **物理隔离**：关键行业专网场景可能部分专用设备，成本高。

**编排与管理（概念）**：切片设计 → 实例化（NFV/MANO 或云编排）→ 运行监控 → 优化/缩容；常涉及运营商 OSS/BSS、切片管理系统等。

## 考点速记

| 考点 | 记忆要点 |
|------|---------|
| 切片定义 | 同一基础设施上多张**逻辑端到端网络** |
| 与 eMBB/uRLLC/mMTC | 典型映射用于记忆，非绝对一一对应 |
| S-NSSAI | **SST + SD**；PDU 会话属性常考 |
| NSSF | **网络切片选择**辅助功能实体 |
| 隔离 | 多为**逻辑隔离**；“纯物理隔离每张切片”表述常为错 |
| E2E | RAN + 传输 + 5GC 协同 |
