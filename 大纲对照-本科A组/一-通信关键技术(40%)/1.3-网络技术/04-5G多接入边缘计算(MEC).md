# 5G多接入边缘计算(MEC)

> 大纲分类：一、通信关键技术 > 三、网络技术 > 5G多接入边缘计算(MEC)  
> 考核要求：掌握

---

## 核心知识点

### 一、MEC（Multi-access Edge Computing）概念

**MEC** 将计算、存储与应用**从中心云下沉**到靠近用户的**网络边缘**（基站侧、接入机房、城域边缘 DC 等），在**无线与固网多接入**场景下为业务提供**更低时延、更高带宽效率、更强本地化处理**能力。

**名称演进提示**：早期文献常写 “Mobile Edge Computing”；ETSI 现统一为 **Multi-access Edge Computing**，强调**不止蜂窝**，也可与 Wi‑Fi 等接入协同。

### 二、MEC 与 5G 结合的典型架构（掌握级）

高层组件通常包括：

- **MEC 平台（MEP）**：托管边缘应用、提供服务注册、API 网关、规则控制等。  
- **MEC 应用（MEC App）**：如本地 CDN、AGV 控制、视频分析、V2X 服务等。  
- **与 5GC 协同**：**UPF 下沉**作为本地数据面锚点；SMF/PCF 等下发策略。  
- **管理与编排**：应用生命周期、镜像部署、资源调度（常与 **NFV/MANO、Kubernetes** 生态结合）。

数据路径上，用户流量经 **UPF** 被**分流**到本地 DN（数据网络）或 MEC 主机，而非一律绕行远端中心。

### 三、UPF 下沉 + 本地分流：ULCL / BP（概念）

3GPP 中常见**多 UPF** 拓扑实现本地卸载：

- **UL CL（Uplink Classifier）**：在上行方向对流量**分类**，将需本地处理的流量导向边缘 UPF/MEC，其余流量仍走中心锚点。  
- **BP（Branching Point）**：在**下行**提供分支，使部分流量可本地投递（教学层掌握“分支/锚点协同”即可）。

**考试关联**：MEC 低时延依赖 **“近端 UPF + 分流规则”**，不是仅靠算力堆叠。

### 四、MEC 平台典型能力（ETSI MEC API 思想）

边缘平台常对外暴露的能力类型包括（名称因厂商而异，掌握类别即可）：

| 能力类别 | 作用 |
|----------|------|
| **无线网络信息服务** | 粗粒度无线侧信息辅助应用自适应（受隐私/策略约束） |
| **位置服务** | 区域/小区级定位辅助（非替代高精度 GNSS/融合定位的全部场景） |
| **带宽管理 / DNS 分流** | 引导流量到最优边缘实例 |
| **用户轨迹 / 接入点变化** | 应用迁移、会话连续性辅助 |

### 五、典型应用场景

- **AR/VR、云游戏**：降低交互时延与回传带宽压力。  
- **工业控制、质检视觉**：数据本地处理，满足时延与安全合规。  
- **V2X**：路侧融合处理、区域态势感知（常与 **C-V2X Uu + PC5** 协同）。  
- **园区/港口/场馆**：本地视频分析与安防。

### 六、易错辨析（对接真题）

- MEC 会带来**本地计算与缓存命中**，但**并非必然减少所有路径上的“传输网负载”**——需看业务是否仍要回传中心云；部分题目把“传输网负载也随之降低”设为**不正确**选项（见下方真题）。

| 与 5G 关键结合 | **UPF 下沉 + 分流（ULCL/BP）** |
| 多接入 | 蜂窝 + Wi‑Fi 等（Multi-access） |
