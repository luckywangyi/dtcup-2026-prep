## 核心知识点

### 一、产品全生命周期阶段（与 IPD / 工程管理对齐）

典型划分（备考可记 **六段**）：

| 阶段 | 核心目标 | 关键活动（概括） |
|------|----------|------------------|
| **概念** | 机会洞察与方向选择 | 市场扫描、技术预研、初步商业论证 |
| **规划** | 定义“做什么、做到什么程度” | 需求基线、目标成本/里程碑、资源与风险预案 |
| **开发** | 把需求变成可制造、可测试的产品 | 详细设计、试制、软硬件联调、文档与可服务性设计 |
| **验证** | 证明产品满足需求与规范 | 各层级测试、缺陷闭环、认证与合规预审 |
| **发布** | 可批量交付市场 | 量产导入、渠道/培训、发布决策、初期保障 |
| **生命周期管理** | 在售与退市阶段持续价值 | 版本维护、EOL 策略、备件与升级、利润与 ROI 复盘 |

**与通信工程传统表述的衔接**（09 笔记）：需求分析 → 方案设计 → 工程施工 → 验收交付 → 运维优化 —— 更偏 **工程项目交付链**；产品全生命周期则覆盖 **从机会到退市** 的完整价值闭环，二者可对照记忆。

### 二、市场调研方法（扩展）

- **定性**：深度访谈、焦点小组、专家研判、竞品走查 —— 适合早期假设验证。  
- **定量**：问卷调查、统计建模、价格敏感度测试 —— 适合规模推断与细分。  
- **现场/数据**：路测与网络 KPI、客服与故障单挖掘、渠道反馈 —— 通信设备商常用。  
- **输出物**：用户画像、场景清单、市场规模与增长率假设、竞争格局与差异化点。

### 三、产品需求分析：OR → IR → AR → DR

（工程化需求分解的常见链条，便于与“需求—设计—验证”闭环对应）

| 层级 | 英文缩写 | 含义（理解要点） |
|------|----------|------------------|
| **OR** | Operator / Organization Requirement | 客户/组织目标与约束（商业、合规、运维） |
| **IR** | Intermediate Requirement | 将 OR 分解为可跟踪的中间需求包 |
| **AR** | Allocated Requirement | 分配到子系统/部件/软件模块的需求 |
| **DR** | Design Requirement | 可验证的设计条目（接口、性能、环境、可靠性） |

**实操要点**：每条 DR 应尽量 **可测试、可追踪**（需求 ID ↔ 测试用例 ↔ 变更记录），避免“口号式需求”。

### 四、产品开发计划制定

- **范围基线**：版本特性列表、不做清单（Out of scope）。  
- **进度**：WBS、关键路径、里程碑（样机、中试、认证、试产）。  
- **资源**：人力/物料/仪表/外协、预算与采购提前期。  
- **风险**：技术风险（指标达不到）、供应链风险、法规与专利（与 09 专利策略联动）。  
- **沟通**：干系人矩阵、评审节奏（与 IPMT/PDT 机制配合，见 `02-工程项目管理.md`）。

### 五、产品测试验证流程

经典分层（名称与划分因企业略有差异，大唐杯常考 **目的与层次**）：

| 层级 | 关注点 | 典型目的 |
|------|--------|----------|
| **单元测试** | 模块/函数/单板 | 尽早发现实现错误，成本低 |
| **集成测试** | 子系统间接口与协议 | 验证互联、时序、异常处理 |
| **系统测试** | 端到端功能、性能、可靠性 | 对照系统规格与认证要求 |
| **验收测试** | 客户合同/标书场景 | 确认可交付，常含现场或准现网环境 |

**黑箱测试**（真题考点）：站在 **用户需求/规格** 外侧，验证 **是否实现约定行为**，而非验证内部实现细节。

### 六、产品安装部署流程（与 5G 工程衔接）

概括：**勘察 → 设计 → 施工 → 调测 → 验收**（09）。

- **勘察**：站址、传输、电源、天面、电磁环境。  
- **设计**：组网与参数（PCI、倾角、功率等）、图纸与 BOM。  
- **施工**：安装、布线、接地与防雷、标签与自检。  
- **调测**：单站、联调、业务拨测、网管接入。  
- **验收**：初验（竣工）、终验（试运行后）；内容常含设备安装、线缆、系统调测、文档交付（09）。

### 七、产品运维优化

- **运维**：监控、告警、变更、问题与配置管理、备件与升级。  
- **优化**：基于 KPI/MR/投诉闭环，驱动版本补丁、参数策略与网络规划调整（与 `05-网络运维及优化.md` 衔接）。

## 考点速记

| 考点 | 记忆要点 |
|------|----------|
| 六阶段 | 概念 / 规划 / 开发 / 验证 / 发布 / 生命周期管理 |
| OR→IR→AR→DR | 组织目标 → 中间需求 → 分配需求 → 可验证设计需求 |
| 黑箱测试目的 | 检验 **客户需求是否实现**（非内部实现路径） |
| 工程交付链 | 需求分析→方案设计→施工→验收→运维优化 |
| 验收 | 初验 + 终验；含设备/线缆/调测/文档 |
| IPMT 生命周期阶段 | 周期长短常考与 **投资回报率** 相关（见真题） |
