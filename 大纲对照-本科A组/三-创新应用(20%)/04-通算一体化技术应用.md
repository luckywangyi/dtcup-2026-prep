## 核心知识点

### 一、背景与必然性（Computing and Communication Convergence）

- **业务驱动**：XR、云游戏、工业控制、自动驾驶、**数字孪生**、大模型推理等要求 **端到端时延** 与 **本地数据处理** 同步满足。  
- **网络演进**：5G/5G-A/6G 从“传数据”走向 **分发算力**，形成 **连接 + 计算 + 存储** 一体化服务。  
- **资源效率**：通过 **边缘卸载** 与 **算力协同**，降低回传带宽与中心云压力。

### 二、算力网络（Computing Power Network）

**定义理解**：以网络为载体，对 **算力资源进行感知、调度、分配**，使用户像获取带宽一样获取 **算力与算法服务**。

**关键能力**：

- **算力感知**：节点算力类型（CPU/GPU/NPU）、负载、时延、能耗。  
- **算力路由**：业务按需选择 **最近/最优** 算力节点（类似路由选路，但度量含 **时延、价格、SLA**）。  
- **算力交易与编排**（产业愿景）：跨域算力调度、异构资源统一封装。

### 三、通算融合架构（概念分层）

```
应用（XR / 孪生 / 质检 / 网联驾驶…）
        ↓
算力服务层（推理/渲染/仿真/大数据）
        ↓
边缘算力 + 区域云 + 中心云（多级协同）
        ↓
确定性连接（5G uRLLC / TSN / 专线）+ 路由与调度
        ↓
终端 / 网关 / RSU / 工业现场设备
```

**与 MEC 的关系**：**MEC（Multi-access Edge Computing）** 是 **通算一体** 在 **接入侧** 的关键落点；**通算一体** 强调 **全网视角** 的算力与连接协同，不限于单站 MEC。

### 四、边缘算力部署形态

- **基站共址 MEC**：路口、园区、工厂车间。  
- **园区/企业边缘机房**：数据驻留与合规。  
- **车载边缘**：高等级自动驾驶 **车端算力 + 路侧 MEC** 协同。  
- **卫星边缘（愿景）**：星地融合下 **星上或信关站侧** 算力（与 `02-星地融合技术应用.md` 联动）。

### 五、算力路由与调度（理解）

- **指标**：端到端时延、丢包、算力队列长度、能耗、价格。  
- **策略**：**延迟敏感任务** 下沉边缘；**批处理/训练** 上送中心或 **区域枢纽**；**故障与拥塞** 时动态迁移。  
- **与 AI 结合**：**08 笔记** — 负荷预测、异常检测、参数推荐，用于 **算力与链路联合优化**。

### 六、典型应用场景

| 场景 | 通算协同要点 |
|------|----------------|
| **XR / 云渲染** | 大带宽上行视觉 + **近端渲染** 降低时延眩晕 |
| **数字孪生** | 现场传感 + **边缘仿真** + 云端长周期优化 |
| **工业 AI 质检** | 产线相机 **本地推理**；结果回传 MES/云平台 |
| **自动驾驶 / 车路协同** | **OBU + RSU MEC** 融合感知与决策 |
| **元宇宙 / 超大并发交互** | 分布式算力 + 内容分发 + 低时延同步 |

**数字孪生**（题库）：在 **工业互联网平台** 等语境中，常考 **设计—制造数据闭环**、**虚实映射**。

### 七、产业发展现状（备考表述）

- **政策**：“东数西算”国家算力枢纽与数据中心集群布局。  
- **运营商**：算力网络、云边协同、5G 定制网 + MEC 行业包。  
- **设备商**：基站内置算力、轻量级 GPU、**AI 推理板**、通感算一体化试验。  
- **标准化**：IETF/IRTF、ETSI MEC、3GPP 5GS 与 **边缘计算互通** 等（了解即可）。
